Model Router

How every LLM call gets routed — what works, what's broken, and why.

What it does

Before every LLM call, the plugin intercepts it and scores the prompt across 4 dimensions. Based on that score it swaps the model to the cheapest one that can handle the task — the user's selected model is the ceiling, not the default.

Fenced code blocks, imports, class/function definitions, multi-file references (3+ files = heavy).

Vision
20%
weight in combined

Image attachments in message or history → 100. Text mentions image keyword → 50. None → 0.

Context
10%
weight in combined

Conversation depth (turns × 3). Tool results in recent history. Context-tree floor. Feedback floor.

Combined formula

combined = text×0.4 + code×0.3 + vision×0.2 + context×0.1 + call_type_offset

What works ✓

WORKS

Multi-dimensional scoring

Text, code, vision, and context are scored independently. Vision always wins priority 1, code wins priority 2 — this prevents a simple "hello + image" from hitting t0 when it needs a vision model.

WORKS

Code specialist bypass

When code score ≥ 50, the router jumps straight to qwen3-coder:480b regardless of the combined text+context score. Massive refactors and multi-file tasks reliably get the code model.

WORKS

Session complexity floor

A RouterSession object tracks the running max complexity. Once a session goes deep (architecture discussion), cheaper models aren't picked for follow-up turns even if they're short.

WORKS

Context tree floor

ContextGraph + ContextTreeBuilder record tool results each turn. The graph's complexity_floor() boosts the context dimension — mid-task tool loops stay on smarter models.

WORKS

Claude provider branch

When the active model is a Claude model (Anthropic provider), a separate ClaudeRouter / ClaudeTiers path is taken — no accidental mixing of Ollama tiers into Claude calls.

WORKS

Kill switch & skip types

HERMES_MODEL_ROUTER=0 disables everything. HERMES_ROUTER_SKIP_TYPES=title,memory lets you pin specific turn types to the user's model instead of routing them.

WORKS

All tier overrides via env vars

Every model slot (HERMES_ROUTER_TIER0 … HERMES_ROUTER_VISION3, HERMES_ROUTER_CODE2) is overridable without touching code.

WORKS

Trivial-task detection

"Hi", "thanks", "ok" patterns score 5 (t0 guaranteed). Phrases like "just rename", "fix typo", "add a comment" deduct 15–20 points — keeps tiny tasks on tiny models.

What doesn't work / known gaps ✗

BROKEN

Vision score weight is wrong

Vision has 20% weight in combined, but vision routing happens before combined is computed (priority 1). The 20% weight never actually influences vision model selection — it's dead weight in the formula.

BROKEN

Code score bypasses combined — even for small tasks

If code ≥ 50, qwen3-coder:480b is picked regardless of context or text scores. A one-line fix with a code block can score code=50 (30 for ``` + 15 for an import = 45, 5 from a keyword) and over-route.

BROKEN

Context tree init silently fails on ImportError

on_session_start catches all exceptions from ContextGraph / ContextTreeBuilder imports and logs at DEBUG. If the tree never initialises, _ctx_builders[session_id] stays empty and the context floor is always 0 — no warning to the user.

BROKEN

Feedback floor is loaded only on turn 0

load_feedback() runs once when session_obj.turns == 0. If feedback is written mid-session it is never reloaded. The feedback floor is stale for the rest of the session.

PARTIAL

Ultra models (deepseek-v3.1, mistral-large-3) are defined but never selected

The docstring mentions U1 and U2 ultra models for the hardest tasks. They appear in the comment but are not in MODELS dict and select_model() never returns them. Dead code.

PARTIAL

call_type_offset values are not documented here

CALL_TYPE_OFFSET is imported from hermes.agent.model_router but the actual numeric values aren't in complexity_matrix.py. The offset is applied to combined score, but you can't see the magnitude without reading a third file.

PARTIAL

Session cap (32) silently evicts active sessions

_clean_old_sessions(keep=32) drops oldest sessions by insertion order. In long-running Hermes processes with many parallel sessions, a session's floor can be reset mid-conversation.

PARTIAL

No ceiling enforcement

The docstring says "user's selected model is the tier-3 ceiling." But if the user's model is t1, the router can still return t3 or c2. There's no guard that caps the routed model to whatever the user has selected.
